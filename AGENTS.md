# AMAR Engine UI Development Guidelines (AGENTS.md)

This project follows a strict **Hardcore Industrial Aesthetic** mixed with **Swiss Minimalism**.

## 🎨 Visual Rules
1. **Color Palette**: 
   - Backgrounds: `#000000` (Pure Black)
   - Borders: `#1E293B` (Slate-800)
   - Accents: `#A7F3D0` (Emerald-200) or `#DEFF9A` (Lime-200)
   - Panel BG: `rgba(2, 2, 2, 0.8)` with backdrop blur.
2. **Density**: Maximum information density. Minimal padding (`p-1`, `p-2`). Use JetBrains Mono for all data fields.
3. **No Decorative Effects**: Avoid rounded corners (max `2px`), shadows, or generic gradients.
4. **Icons**: Use industrial/technical icons from `lucide-react`.

## 🧠 AI Copilot Integration
1. **Model Awareness**: The app supports multiple Gemini models. When adding features to the AI Copilot (`AICopilot.tsx`), ensure compatibility with function calling.
2. **Command Consistency**: Natural language commands should predictably map to state changes in `useEngineStore.ts`.
3. **Canvas Logic**: The `WorkflowStudio` is coordinate-aware. New nodes generated by AI should be positioned logically (e.g., center of viewport or staggered relative to existing nodes).

## 🛠 Tech Stack Constraints
- **State**: Zustand (`useEngineStore.ts`)
- **Canvas**: Konva (`react-konva`)
- **Animations**: motion (`motion/react`)
- **Icons**: Lucide React
- **LLM SDK**: `@google/genai` (Gemini)

## 📐 Component Structure
- Always separate UI components from business logic.
- Use `PanelHeader` for all modular panels.
- Maintain the "Workstation" feel with `react-resizable-panels`.
